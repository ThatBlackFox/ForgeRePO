MTL-ForgeRePO – Multi-Task-Learner using Forget-Recall & Pareto Optimization

Field of the Invention

The invention pertains to Artificial Intelligence and Computational Drug Discovery, focusing on Continual Learning frameworks for Molecular Property Prediction using Large Language Models (LLMs). It introduces a Multi-Task Learner (MTL) system based on the Forget-Recall (ForgeRe) principle and Pareto Optimization (PO) to mitigate Catastrophic Forgetting (CF) during sequential learning from SMILES-based molecular representations.

Invention Overview

The MTL-ForgeRePO framework discloses a Continual Learning (CL) system and method for molecular property prediction, enabling LLMs to learn new molecular data without losing prior task knowledge. Unlike replay-based CL methods that are computationally heavy and disrupt inter-task relations, this invention performs selective forgetting before adaptive recall — inspired by the human brain’s memory mechanism.

By integrating Pareto Optimization to balance the stability–plasticity trade-off and a ChemBERTa-based transformer with hierarchical gradient aggregation, the model eliminates the need for replay buffers while achieving stable, multi-task generalization across datasets such as BBBP, Bitter, and Sweet.

Datasets

The framework utilizes three curated SMILES-based molecular datasets to facilitate multi-task continual learning across taste and permeability prediction tasks:

  1. Sweet or Not Dataset
  2. Bitter or Not Dataset
  3. Blood-Brain Barrier Permeable or Not (BBBP) Dataset

Each dataset is preprocessed using standardized SMILES canonicalization and molecular tokenization to ensure cross-task consistency and facilitate transfer learning within the continual-learning framework.

System Architecture

The framework comprises four primary modules:

  1. Data & Pre-processing Module – Handles SMILES input, molecular tokenization, and embedding generation.
  2. Multi-Task Learner – Integrates three stacked ChemBERTa models trained on BBBP, Bitter, and Sweet datasets (Episodes A, B, C) with positional and token embeddings, multi-head attention, and feed-forward layers optimized via Pareto fronts.
  3. Model Evaluation Module – Implements oEWC, DER++, CPR, and ER to evaluate continual-learning stability and visualize performance metrics.
  4. Application Module – A computer-implemented application built with FastAPI, featuring Connect, Train, and Inference modules for real-time molecular prediction, visualization, and repository management. Deployed effectively on Android devices.

This system achieves superior predictive accuracy and reduced forgetting across molecular datasets, providing a scalable and interpretable continual-learning solution for drug discovery, molecular design, and pharmacokinetic screening.
